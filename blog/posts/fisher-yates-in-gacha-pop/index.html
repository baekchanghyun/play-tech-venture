안녕하세요, 플텍에서 앱의 기본 뼈대를 만드는 일을 하고 있는 민준입니다. 하윤님이 ”🎲 기획서만 쓰던 기획자가 게임을 만들어 봤습니다”에서 주사위 던전 게임 이야기를 해주셨는데요. 거기서 “설계서에 빠진 건 구현에서도 빠진다”는 교훈이 나왔잖아요. 오늘은 그 반대 사례를 하나 가져왔습니다. 설계서에 “공정하게 섞는다”라고 한 줄 적어놓은 걸 보고 정말 공정한 로직을 만들기 위해 끝까지 파고들어본 경험담입니다.

캡슐을 섞어야 합니다

가챠팝(Gacha Pop)은 2~8명이 돌아가며 캡슐을 하나씩 뽑는 파티 게임인데요. 핵심은 “누가 당첨을 뽑을지 모른다”는 긴장감입니다. 이 긴장감이 성립하려면 캡슐이 정말로 공정하게 섞여야 합니다. 먼저 뽑는 사람이 유리하면 안 되고, 나중에 뽑는 사람이 불리하면 안 됩니다.

앱의 기본 뼈대를 담당하는 제가 이 셔플 로직을 맡았는데요. 결론부터 말씀드리면, Fisher-Yates 셔플이라는 알고리즘을 사용했습니다. 왜 이걸 선택했는지, 그리고 어떻게 동작하는지 이야기해보겠습니다.

일단 “sort(() => Math.random() - 0.5)“는 안 됩니다

JavaScript에서 배열을 섞고 싶을 때, 흔히 이런 코드를 보게 되는데요.

array.sort(() => Math.random() - 0.5);

간결해서 좋아 보이지만, 이 방식은 균일하지 않습니다. sort() 함수의 비교 함수는 “a와 b 중 누가 앞에 와야 하는가”를 일관되게 답해야 하는데, 랜덤 값을 넣으면 같은 쌍에 대해 매번 다른 답을 하게 됩니다. 정렬 알고리즘(보통 TimSort)의 내부 비교 순서에 따라 특정 위치가 편향되는데요.

…아, 잠깐. TimSort의 merge 과정에서 비교 방향이 어떻게 편향을 만드는지까지 설명하려면 글이 너무 길어지겠네요. 핵심만 말씀드리면: 이 방법은 모든 순열이 동일한 확률로 나오지 않습니다. 그래서 쓰면 안 됩니다.

Fisher-Yates는 뭔가요

Fisher-Yates 셔플(정확히는 Knuth가 개선한 현대 버전)은 이런 알고리즘입니다.

for (let i = array.length - 1; i > 0; i--) {
  const j = Math.floor(Math.random() * (i + 1));
  [array[i], array[j]] = [array[j], array[i]];
}

배열의 끝에서부터 시작해서, 각 위치에 대해 “0부터 현재 위치까지 중 아무 위치나 하나 골라 교환”합니다.

이게 왜 공정한지를 수학적으로 증명하면… 각 단계에서 아직 결정되지 않은 원소들 중 하나를 균일하게 선택하기 때문에, 결과적으로 N! 가지 순열이 모두 동일한 확률 1/N!로 나타나게 되는데요. N개 원소에 대해 첫 번째 원소가 특정 위치에 올 확률이 1/N이고, 두 번째 원소가 남은 위치에 올 확률이 1/(N-1)이고, 이걸 전부 곱하면 각 순열의 확률이 1/N!이 되는… 그러니까…

음, 좀 멀리 갔습니다. 간단히 말하면 “모든 순열이 정확히 같은 확률로 나온다”는 게 수학적으로 증명된 알고리즘입니다. 시간복잡도는 O(N)이고, 제자리(in-place)에서 동작합니다.

실제 코드

가챠팝의 실제 구현은 이렇게 생겼는데요.

export function shuffleCapsules(contents: CapsuleContent[]): Capsule[] {
  const capsules: Capsule[] = contents.map((content, i) => ({
    id: i,
    color: CAPSULE_COLOR_ORDER[i % CAPSULE_COLOR_ORDER.length],
    content,
  }));

  // Fisher-Yates 셔플
  for (let i = capsules.length - 1; i > 0; i--) {
    const j = Math.floor(Math.random() * (i + 1));
    [capsules[i], capsules[j]] = [capsules[j], capsules[i]];
  }

  return capsules;
}

먼저 각 내용물에 색상을 할당해서 Capsule 객체를 만든 다음, Fisher-Yates로 섞습니다. 색상은 원래 순서대로 고정이고, 위치만 바뀌는 거죠.

근데 정말 공정한지 어떻게 확인하죠

코드를 읽고 “Fisher-Yates니까 공정하겠지”라고 넘어갈 수도 있지만, 저는 좀 불안해서 직접 검증했습니다. 10,000회, 5,000회, 60,000회 시뮬레이션을 돌리는 장기 밸런스 테스트를 작성했는데요.

핵심 테스트 하나만 소개하면:

// 3명 게임을 60,000회 반복하면, 3! = 6가지 순열이 각각 약 10,000회 등장해야 함
// 카이제곱 검증: χ² < 15.09 (자유도 5, 유의수준 0.01)

3개 캡슐의 가능한 배치는 ABC, ACB, BAC, BCA, CAB, CBA의 6가지인데요. 60,000회 셔플하면 각각 약 10,000회씩 나와야 합니다. 여기에 **카이제곱 검정(chi-squared test)**을 적용했습니다. 각 순열의 관찰 빈도와 기대 빈도(10,000)의 차이를 제곱해서 합산한 값이 χ² 통계량이 되고, 이 값이 자유도 5, 유의수준 0.01 기준인 15.09보다 작으면 “편향 없음”으로 판단합니다. 결과는 통과였습니다.

비복원 추출의 교환가능성(exchangeability)이라는 성질 덕분에, k번째로 뽑는 사람이 당첨 캡슐을 뽑을 확률은 순서에 관계없이 항상 1/N입니다. 1번째든 8번째든 확률은 같습니다. 직관에 반하지만, 조건부 확률을 순서대로 전개해보면 깔끔하게 증명됩니다. 1번째 사람이 당첨을 뽑지 않을 확률 (N-1)/N 아래에서, 2번째 사람이 당첨을 뽑을 확률은 1/(N-1)이고, 이 둘을 곱하면 결국 1/N이 됩니다. k번째도 마찬가지입니다.

더 알고 싶으시다면

이 글에서 다룬 내용을 깊이 공부하고 싶으시다면, 아래 키워드를 참고해주세요.

  • Fisher-Yates shuffle (Knuth shuffle): 위키피디아 문서가 잘 정리되어 있습니다. “inside-out” 변형도 재미있습니다
  • sort 기반 셔플의 편향 문제: Mike Bostock의 “Fisher-Yates Shuffle” 시각화 글이 유명합니다. 편향이 눈에 보입니다
  • 카이제곱 검정 (Chi-squared test): 관찰값과 기대값의 적합도를 검증하는 통계 기법입니다. 자유도와 유의수준 개념을 함께 보시면 됩니다
  • 교환가능성 (Exchangeability): 비복원 추출에서 “순서가 확률에 영향을 주지 않는다”는 성질입니다. 확률론 입문서에서 다뤄집니다

가챠팝에서 캡슐을 뽑을 때 “먼저 뽑으면 유리한 거 아니야?”라고 물어보시는 분이 계시다면, 이 글을 보여주시면 됩니다. 수학적으로 동일합니다.

가챠팝을 직접 해보고 싶으신 분은 아래 링크에서 플레이할 수 있습니다. 캡슐이 정말 공정하게 섞이는지, 직접 확인해보세요.
