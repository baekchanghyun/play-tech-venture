안녕하세요, 플텍에서 UI 개발을 담당하고 있는 시우입니다. 리듬 분석기(Rhythm Analyzer)를 만들면서 한 가지 골치 아픈 문제가 있었습니다. 메트로놈을 틀어놓고 BPM 슬라이더를 조절하면, 소리는 바뀌기 전 속도로 계속 재생되는 거였습니다. setInterval 안에서 React state가 업데이트를 따라가지 못하는 문제였는데요.

원인은 금방 찾았지만, 타이머와 React 렌더링을 어떻게 깔끔하게 조화시킬지가 쉽게 떠오르지 않았습니다. 며칠 동안 머릿속에 맴돌던 이 문제가, 퇴근 후 미니벨로를 타고 한강 자전거길을 달리다가 문득 풀렸습니다. 페달을 밟으면서 “ref로 동기화하면 되잖아”라는 생각이 스친 순간, 자전거를 세우고 폰에 메모를 남겼습니다.

이 글에서는 그 고민과 해결 과정을 처음부터 풀어보려고 합니다.

리듬 분석기를 직접 사용해보고 싶으신 분은 아래 링크에서 체험할 수 있습니다.

리듬 분석기(Rhythm Analyzer)

리듬 앱에서 타이밍이 왜 중요한가

리듬 분석기는 사용자가 탭하는 간격을 측정해서 BPM과 안정성을 분석하는 앱입니다. 메트로놈 모드에서는 비트에 맞춰 탭하면 ±50ms 이내가 “perfect”, ±100ms 이내가 “good”으로 판정됩니다.

50ms라는 숫자가 얼마나 짧은지 감이 안 오실 수 있는데요. 눈을 한 번 깜빡이는 데 약 150ms가 걸립니다. 그 1/3 수준의 정밀도가 필요한 겁니다. React의 상태 관리가 이 정밀도를 방해할 수 있다는 걸 개발 중에 알게 됐습니다.

setInterval 안에서 state가 멈추는 문제

처음에는 평범하게 useState로 구현했습니다. 메트로놈을 시작하면 setInterval로 비트를 세고, 탭이 들어오면 현재 BPM과 시작 시간을 읽어서 오프셋을 계산하는 구조였는데요.

문제는 setInterval의 콜백이 생성 시점의 state를 캡처한다는 것이었습니다. React의 클로저 특성상, 콜백 안에서 읽는 targetBpm은 interval이 만들어진 순간의 값이었습니다. 사용자가 BPM 슬라이더를 움직여도 콜백은 이전 값으로 동작했습니다.

서두에서 말씀드린 것처럼 이 문제를 며칠 동안 곰곰이 생각해봤는데요. React가 state를 스냅샷으로 관리한다는 기본 원리를 떠올리면 당연한 동작이었습니다. 렌더링마다 새로운 state 값이 생기지만, 이미 등록된 타이머 콜백은 이전 렌더의 클로저에 갇혀 있으니까요.

useRef로 클로저를 우회하다

해결책은 useRef였습니다. ref는 렌더링 사이에서 값이 유지되면서도, .current에 접근하면 항상 최신 값을 읽을 수 있습니다.

const targetBpmRef = useRef<number>(120);
const isPlayingRef = useRef<boolean>(false);
const startTimeRef = useRef<number>(0);

useEffect(() => {
  targetBpmRef.current = targetBpm;
}, [targetBpm]);

useState로 UI를 업데이트하되, 타이머 콜백에서 읽어야 하는 값은 전부 ref에 동기화합니다. setInterval 안에서는 targetBpmRef.current를 읽으면 항상 최신 BPM을 볼 수 있습니다.

탭 시점의 타임스탬프에는 performance.now()를 사용했습니다. Date.now()는 밀리초 단위지만 시스템 시계 보정에 영향을 받을 수 있고, performance.now()는 단조 증가하면서 마이크로초 정밀도를 보장합니다. 리듬 앱에서는 이 차이가 중요합니다.

오디오 파일 없이 클릭음 만들기

메트로놈 소리도 재미있는 부분이었습니다. mp3 파일을 재생하는 대신 Web Audio API로 소리를 직접 합성했는데요. OscillatorNode로 사인파를 만들고, GainNode로 30ms 만에 급격히 소멸시키면 짧고 선명한 클릭음이 됩니다.

const oscillator = audioContext.createOscillator();
oscillator.type = "sine";
oscillator.frequency.value = isAccent ? 1500 : 1000;

const gainNode = audioContext.createGain();
gainNode.gain.setValueAtTime(1, now);
gainNode.gain.exponentialRampToValueAtTime(0.001, now + 0.03);

강박에는 1500Hz, 일반 박에는 1000Hz를 사용해서 4/4 박자의 첫 박을 구분할 수 있게 했습니다. exponentialRampToValueAtTime이 핵심인데, 선형이 아니라 지수적으로 감쇠하기 때문에 사람 귀에 자연스러운 “틱” 소리가 됩니다.

SVG로 BPM 게이지 그리기

BPM을 시각적으로 보여주는 원형 게이지도 SVG의 strokeDashoffset으로 구현했습니다. 원의 둘레(2πr)를 strokeDasharray로 설정하고, 채울 비율만큼 strokeDashoffset을 줄이는 원리입니다.

const CIRCUMFERENCE = 2 * Math.PI * 72;
const offset = CIRCUMFERENCE * (1 - bpm / 240);

한 줄의 계산으로 0~240 BPM 범위를 원형 프로그레스에 매핑할 수 있습니다. arc 경로를 직접 계산할 필요가 없어서 깔끔합니다.

정리하며

리듬 분석기를 만들면서 React의 렌더링 모델을 다시 한번 깊이 생각하게 됐습니다. state는 렌더링의 스냅샷이고, ref는 렌더링을 관통하는 가변 상자입니다. 평소에는 이 차이가 크게 느껴지지 않지만, 밀리초 단위의 정밀도가 필요한 앱에서는 결정적인 차이를 만듭니다.
